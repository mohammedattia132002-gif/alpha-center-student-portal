## 1. ملخّص تنفيذي

البوابة **تعمل وتبني بنجاح** (`tsc --noEmit` يمرّ نظيفاً). التدفق الأساسي (دخول → رئيسية → حضور → مدفوعات → درجات → امتحانات) مبنيّ على بيانات حقيقية من Supabase.

لكن للوصول لنسخة نهائية موثوقة تحتاج معالجة **أربع فئات**:
1. **كود ميت** يضخّم الحزمة ويربك الصيانة (4 مكوّنات يتيمة + كتل ميتة داخلية).
2. **bug صامت** في طبقة التحفيز يعطّل ميزة كاملة.
3. **نص عربي مشوّه (mojibake)** يظهر للمستخدم النهائي.
4. **بيانات وهمية/محاكاة معروضة كأنها حقيقية** (دفع، تحليل ذكي، إشعارات) — مخاطرة ثقة/نزاهة.

## 3. سجل المشاكل (Issue Register)

### 🔴 حرجة

**C1 — مكوّنات يتيمة (dead code)**
- الملفات: `Header.tsx`, `OnlineExams.tsx`, `AIAnalysis.tsx`, `GlobalEnhancements.tsx`.
- غير مستوردة في `App.tsx`؛ مراجع ذاتية فقط.
- الأثر: تضخيم، التباس صيانة، أكبر تجمّع بيانات وهمية.

**C2 — bug صامت: حقل `percentage` غير موجود**
- الملف: `GlobalEnhancements.tsx:83-86, 120`.
- يقرأ `g.percentage` بينما `GradeRecord` يملك `score/maxScore` فقط (راجع `types.ts:55-68`).
- الأثر: `undefined >= n` = `false` دائماً → XP/أوسمة الدرجات = 0 دائماً. الميزة معطّلة صامتة.

**C3 — نص عربي مشوّه (mojibake)** يراه المستخدم
- الملف: `supabaseClient.ts` أسطر ~30, 136, 180, 292, ~1292-1294, 1353-1355.
- أمثلة: رسالة "فشل الاتصال بـ Supabase"، "غير محدد"، تعليمات الامتحان، أسماء افتراضية.
- الأثر: نص "ط§ظ„..." أمام الطالب في رسائل الخطأ والتعليمات.

**C4 — دفع وهمي يدّعي بوابات حقيقية**
- الملف: `Payments.tsx:194-205` (setTimeout 1800ms) + نص يذكر "Stripe / فوري / تشفير".
- الأثر: خداع المستخدم؛ مخاطرة قانونية/ثقة. يجب إما ربط بوابة حقيقية أو تصريح واضح بأنه عرض تجريبي.

### 🟡 متوسطة

**M1 — Attendance heatmap يعرض أقدم السجلات**
- `Attendance.tsx:87-97` يأخذ `records[i]` (أول 28) ثم `.reverse()` بدل أحدث 28.

**M2 — صيغة معدل الالتزام بوزن سحري**
- `Attendance.tsx:84` يستخدم `lateCount*0.72` غير موثّق ويحتسب "معذور" حضوراً كاملاً.

**M3 — نموذج الانضمام يهمل حقولاً**
- `AuthScreens.tsx:99-104` لا يُرسل `academicStage`/`academicGroup`/`gender` رغم ملء المستخدم لها.

**M4 — snapshot ضخم (688KB) + fallback صامت**
- `data/student-snapshots.json` يُحزَم في كل بناء ويُدمج دون مؤشّر "بيانات قديمة/غير متصل".

**M5 — Dashboard: كتل ميتة + خطأ حسابي**
- `Dashboard.tsx:75-98` قائمة todo وبيانات streak وهمية غير معروضة + `*105` (يفترض 100).

**M6 — إمكانية الوصول (accessibility)**
- حقول بحث/إدخال بلا `<label>`؛ نوافذ منبثقة بلا `role="dialog"`/حبس تركيز؛ Toast بلا `aria-live`.

### 🟢 منخفضة
- `Grades.tsx` تعليق افتراضي عن "أمن البرمجيات" غير مناسب لطالب ثانوي.
- `Toast.tsx` `duration` معرّف بلا إغلاق تلقائي؛ `useEffect` فارغ.
- `EmptyState.tsx` ألوان `text-white/40` غير مقروءة فوق خلفية فاتحة.
- ملفات `.tmp_*` و`inspect_db.js` وأصناف Tailwind غير قياسية (`indigo-650`) — تحقق من الإعداد.

## 4. خطة التنفيذ المرحلية

> كل مرحلة قابلة للإقرار والتنفيذ منفصلة. لا تبدأ مرحلة قبل إقرارها.

### المرحلة 0 — تنظيف آمن (لا يمسّ السلوك)
- **[C1]** حذف المكوّنات اليتيمة الأربعة (أو نقلها لمجلد `_archive/` إن رُغب في الاحتفاظ).
- **[C3]** إصلاح كل سلاسل mojibake في `supabaseClient.ts` بنص عربي سليم.
- **[M5]** حذف كتل Dashboard الميتة (todo/streak) وإصلاح `*105`.
- **معيار الخروج:** `tsc --noEmit` يمرّ، حجم الحزمة أصغر، لا تغيّر مرئي في الشاشات الخمس.

### المرحلة 1 — نزاهة البيانات (إزالة الوهم)
- **[C4]** قرار الدفع: (أ) ربط بوابة حقيقية، أو (ب) تحويله لوضع "تجريبي" مصرّح به بوضوح وإزالة ادعاءات Stripe/التشفير.
- **[M1]** إصلاح heatmap ليأخذ أحدث 28 يوماً.
- **[M2]** توثيق/تبسيط صيغة المعدل (اعتماد `useStudentStats` الموجودة كمصدر وحيد).
- **[M3]** إرسال الحقول المفقودة في نموذج الانضمام + تصفيرها بعد النجاح.
- **معيار الخروج:** لا رقم/ادعاء معروض للمستخدم إلا وله مصدر بيانات حقيقي.

### المرحلة 2 — إمكانية الوصول والتجاوب
- **[M6]** ربط `<label>` بالحقول، إضافة `role`/`aria-*`، حبس تركيز النوافذ، `aria-live` للـToast والمؤقّت.
- إصلاح تباين `EmptyState`.
- **معيار الخروج:** تنقّل كامل بلوحة المفاتيح؛ فحص a11y أساسي نظيف.

### المرحلة 3 — الأداء وتجربة الأوفلاين
- **[M4]** تحميل كسول أو إزالة snapshot 688KB + مؤشّر "آخر تحديث/غير متصل" صريح.
- مراجعة إنشاء `AudioContext` المتكرر (Dashboard/Attendance/Grades) → مثيل واحد مشترك.
- **معيار الخروج:** حزمة أخف؛ لا staleness صامت.

### المرحلة 4 — مواءمة المزامنة (تعتمد على SYNC_AUDIT)
- **[C3-sync]** توجيه كتابات البوابة (`payInvoice`/`createExcuse`/`insertGrade`/exam results) عبر `apply_changes` بدل الكتابة المباشرة.
- **[H5]** توحيد الطوابع الزمنية (epoch-ms) عبر كل الكتابات.
- **تحذير:** لا تبدأ قبل تجميد عقد الـ RPC في الخلفية (المرحلة 0 من SYNC_AUDIT).

### (اختياري) المرحلة 5 — إعادة إحياء الميزات اليتيمة
- إن رُغب لاحقاً: ربط تحليل حقيقي/تحفيز بعد إصلاح `percentage` وربطه ببيانات فعلية.
